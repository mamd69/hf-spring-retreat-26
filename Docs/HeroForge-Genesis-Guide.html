Section 01

What Is Genesis?

Genesis is an operating system for your business. It analyzes your business, plans improvements, builds the solutions, deploys them, and then monitors and self-improves over time. It is not a chatbot. It is not a single tool.

The Three-Layer Knowledge Engine

Most AI tools are static — you ask a question, get an answer, and the AI forgets you. Genesis is built on three layers that work together:

🔍

Vector Database

Finds information by meaning, not just keywords. Ask about "PTO policy" and it finds "vacation days."

A librarian who understands what you mean, not just what you said.
🕸

Graph Database

Understands relationships — org charts, coding rules, regulatory hierarchies.

A map that shows how everything connects to everything else.
🧠

Graph Neural Network

Learns and improves from every interaction, autonomously optimizing toward your goals.

An employee who gets better every day without being told how.

That third layer — the GNN — is the key differentiator. AI models like Claude and GPT do not learn from your interactions. They are born smart but have no memory. The GNN actually learns. Set a goal like "improve lead conversion rate" and it optimizes toward that goal over time, without human intervention.

Section 02

Setting Up Your Tools

You need four things: a GitHub account, a Codespace, Claude Code, and Ruflo. This takes about 30 minutes.

1

Create a GitHub Account

GitHub is where your project files live. It is free.

  1. Go to github.com in your web browser.
  2. Click Sign up in the top-right corner.
  3. Enter your email, create a password, and choose a username.
  4. Complete any verification puzzles and enter the email verification code.
Pricing Note
GitHub's free tier is generous for basic usage. Consider a paid account later for larger machine types or more storage.
2

Create Your First Repository

A repository ("repo") is a project folder that stores your files.

  1. Click the + icon in the top-right corner and select New repository.
  2. Repository name: Type a name (example: my-genesis-project).
  3. Visibility: Choose Private to keep your data secure.
  4. Check Add README.
  5. Click Create repository.
3

Open a GitHub Codespace

A Codespace is a complete development environment running in your browser. No software to install.

  1. From your repo's main page, click the green Code button.
  2. Click the Codespaces tab.
  3. Click Create codespace on main.
  4. Wait 1-2 minutes. You will see an editor with a file sidebar, code area, and terminal panel.
Tip
If you have VS Code Desktop, you can open a Codespace from there: Start > Connect to... > Connect New Codespace.
4

Install Claude Code

Claude Code is your AI-powered coding assistant. It runs in the terminal.

Terminal 
curl -fsSL https://claude.ai/install.sh | bash

Wait 1-2 minutes for installation to complete.

5

Create Shortcuts

These save you from typing long commands. Paste into your terminal:

Terminal 
echo -e "alias dsp='claude --dangerously-skip-permissions'\nalias dsp-c='claude --dangerously-skip-permissions -c'" >> ~/.bashrc

Then activate them:

Terminal 
source ~/.bashrc
ShortcutWhat It Does
dspStarts Claude Code without permission prompts (required for agent workflows)
dsp-cSame, but resumes your last session
On Mac Desktop
Replace ~/.bashrc with ~/.zshrc in both commands above.
6

Install Ruflo

Ruflo coordinates multiple AI agents simultaneously.

Terminal 
curl -fsSL https://cdn.jsdelivr.net/gh/ruvnet/claude-flow@main/scripts/install.sh | bash -s -- --full
7

Start Claude Code & Authenticate

  1. Type dsp in the terminal and press Enter.
  2. Choose display mode: 1 for dark, 2 for light.
  3. Choose account type: 1 for Claude Max, 2 for Pro + API billing.
  4. Click the authorization link that appears.
  5. Click Authorize on the Anthropic website.
  6. Paste the key back into your terminal.
Known Issue
You may see an error the first time you click Authorize. Go back to your Codespace, click the link again while holding Cmd (Mac) or Ctrl (Windows), and authorize again.
Setup Complete
You now have everything you need.

File TypeHow It Processes

Tip
Start with transcripts — they are the richest source of context. Financial CSVs are surprisingly useful too: each transaction becomes a searchable sentence like "$1,499 payment from Jane Doe for Mastermind Pro on Mar 8, 2026."

Troubleshooting

"Search results don't seem relevant" — The semantic embedding engine did not load. Look for ONNX embedder ready: 384d semantic embeddings at startup. Make sure the ruvector/ directory exists.

Section 05

Elevating Your Knowledge Base to World-Class

Go from "we uploaded our documents" to "we have the most complete understanding of our domain that exists." Eight sub-phases, each with a quality gate.

Phase 2.1 — Domain Scoping

Define Your Boundaries

Define what your knowledge base covers so it does not miss critical areas or become bloated.

Paste into Claude Code 
Analyze my knowledge base and define the domain scope. For my business in
[YOUR INDUSTRY], document:

1. A formal definition of our domain and its boundaries
2. Adjacent fields that overlap with ours
3. Sub-disciplines within our domain
4. Historical context — how this field got to where it is today
5. Current debates and open questions in the industry

Save to docs/kb-enhancement/domain-scope.md
Phase 2.2 — Perspective Expansion

Map Every User and Question

The front desk manager asks different questions than the CFO. Map them all.

Paste into Claude Code 
Based on the domain scope in docs/kb-enhancement/domain-scope.md, map out:

1. At least 5 types of users who will query this knowledge base
2. At least 50 types of questions these users would ask
3. Use cases for each user type
4. Edge cases — unusual but important scenarios
5. Interaction patterns — how different users search differently

Save to docs/kb-enhancement/perspective-map.md
Phase 2.3 — Expert Discovery

Pull in Industry Knowledge

Your documents capture what YOUR business knows. This step captures what the INDUSTRY knows.

Paste into Claude Code 
Based on my domain scope, identify the top 100 experts across these categories:

- Academic researchers and published authors
- Practitioners and clinicians (if applicable)
- Industry leaders and conference speakers
- Authors of key books and frameworks
- Emerging voices and innovators

For each expert, extract their key contributions, frameworks, and insights.
Cross-reference insights across experts to identify consensus and disagreement.
Target: 500+ content items yielding 1000+ cross-referenced insights.

Save expert profiles to docs/kb-enhancement/experts/
Save extracted insights to docs/kb-enhancement/insights/
Phase 2.4 — Completeness Audit

Find What's Missing

Paste into Claude Code 
Audit the current knowledge base for gaps. Check across these dimensions:

1. Coverage gaps — topics in our domain scope that have no content
2. Depth gaps — topics covered superficially that need more detail
3. Perspective gaps — viewpoints not represented
4. Temporal gaps — outdated information or missing recent developments
5. Structural gaps — areas where relationships between topics are missing

Rate each gap by importance (critical, high, medium, low).
Identify at least 30 gaps total.

Save to docs/kb-enhancement/completeness-audit.md
Phase 2.5 — Gap Filling

Fill Every Hole

Paste into Claude Code 
Using the completeness audit in docs/kb-enhancement/completeness-audit.md,
fill all critical and high-importance gaps. For each gap:

1. Research authoritative sources
2. Extract relevant knowledge
3. Integrate into the existing knowledge base structure
4. Verify accuracy against multiple sources

Start with critical gaps first. Document sources for everything.
Save new content to the appropriate knowledge base locations.
Save a gap-filling log to docs/kb-enhancement/gap-filling-log.md
Phase 2.6 — Structure & Organize

Maximum 9 Top-Level Categories

Paste into Claude Code 
Organize the full knowledge base into a clear structure following these rules:

1. Maximum 9 top-level categories (cognitive limit — more becomes unnavigable)
2. Each category follows MECE principles (no overlaps, no gaps)
3. Unlimited depth within categories
4. Every piece of content has a clear home

Produce a visual map of the structure and save to
docs/kb-enhancement/structure-map.md
Phase 2.7 — Depth Verification

No Empty Nodes

Paste into Claude Code 
Review every leaf node in the knowledge base structure. Each must contain:

1. A clear title
2. Substantive content (not just a placeholder)
3. Expert source attribution
4. Source URL or reference
5. Confidence score (how certain is this information?)
6. Metadata (date, category, related topics)

Identify any leaf nodes that are empty or placeholder-only and fill them.
Save the audit results to docs/kb-enhancement/depth-audit.md
Phase 2.8 — Quality Scoring Loop

Score, Improve, Repeat Until 98+

Paste into Claude Code 
Score the knowledge base on these 6 criteria (each weighted):

1. Completeness (25%) — Does it cover the full domain scope?
2. Accuracy (25%) — Is the information correct and well-sourced?
3. Attribution (15%) — Can every claim be traced to a source?
4. Structure (15%) — Is it logically organized and navigable?
5. Depth (10%) — Are topics covered with sufficient detail?
6. Currency (10%) — Is the information up to date?

Produce an overall score out of 100. For any criterion below 90%, identify
specific improvements. Implement the improvements and re-score.

Repeat until the overall score is 98 or above.

Save scoring reports to docs/kb-enhancement/quality-scores/

Don't implement yet. Put planning files in:

- docs/ddd/
- docs/plan/implementation-plan.md

Best Practices

  • Don't paste large text into Claude Code. Put content into files and reference them by path instead.
  • Use multiple terminal windows when tasks need to run in parallel.
  • Save and commit frequently. This creates save points you can return to.
  • Read error messages carefully. They usually tell you exactly what went wrong.
  • Be specific. "Fix the dashboard" is vague. "The revenue chart shows last month's data instead of this month's" is specific.
  • Keep your laptop awake during long builds. Run caffeinate in a separate terminal.
  • Your expertise is essential. The AI does not know your business the way you do. Review, correct, and direct.
